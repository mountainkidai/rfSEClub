# Phase 0.8 — Arithmetic Logic Unit (ALU)

## Introduction

In the previous chapters, we built the foundations of a computer:

```text
Information -> Bits -> Boolean Logic -> Transistors -> Logic Gates -> Digital Circuits -> Registers
```

At this point, our computer can:

- Store information.
- Remember information.

But it still cannot actually compute.

Imagine a human with a notebook.

The notebook can store numbers.

But the notebook itself cannot perform:

```text
5 + 3
```

or

```text
10 > 4
```

or

```text
7 AND 3
```

Something must perform the calculations.

`Inside a CPU, that component is called the:`

> `Arithmetic Logic Unit (ALU)`

The ALU is the calculator of the processor.

Every addition, subtraction, comparison, and logical operation eventually passes through the ALU.

---

## What Is An ALU?

ALU stands for:

```text
Arithmetic Logic Unit
```

Break it down:

### Arithmetic

Mathematical operations:

```text
+ - × ÷
```

### Logic

Boolean operations:

```text
AND OR XOR NOT
```

### Unit

`A physical circuit inside the CPU.`

---

## The Big Picture

Think of the CPU as a factory.

```text
+----------------------+
|         CPU          |
|                      |
|  Registers           |
|      ↓               |
|  ALU                 |
|      ↓               |
|  Result Register     |
+----------------------+
```

Registers hold data.

The ALU processes data.

The result goes back into registers.

---

## Why The ALU Exists

Suppose your program says:

```rust
let x = 5;
let y = 3;
let z = x + y;
```

The CPU must answer:

```text
5 + 3 = ?
```

The ALU performs this calculation.

Without an ALU:

```text
CPU = Memory Only
```

The processor could remember numbers but never manipulate them.

```text

           +-------------+
```

Inputs:

```text
A = 5

```

Operation:

```text

```

Output:

```text
8
```

## Arithmetic Operations

The first responsibility of the ALU is arithmetic.

---

### Addition

Example:

```text
A = 5
B = 3
Result = 8
```

Binary:

```text
0101
0011
----
1000
```

Internally this is performed using Full Adders.

---

### How Addition Actually Happens

The ALU does not know mathematics.

It only knows logic gates.

Example:

```text
Bit 0 -> Full Adder
Bit 1 -> Full Adder
Bit 2 -> Full Adder
Bit 3 -> Full Adder
```

ASCII representation:

```text
A3   A2   A1   A0
|    |    |    |
v    v    v    v
+--+ +--+ +--+ +--+
|FA| |FA| |FA| |FA|
+--+ +--+ +--+ +--+
 |    |    |    |
 +----+----+----+--> Result
```

FA = Full Adder

Many Full Adders connected together become an arithmetic unit.

---

### Subtraction

Example:

```text
10 - 3
```

Result:

```text
7
```

```text
1010
```

The second responsibility of the ALU is logic.

### AND Operation

Example:

```text

```

```text
1 AND 1 = 1
1 AND 0 = 0
0 AND 1 = 0
0 AND 0 = 0
```

Result:

```text
1000
```

---

### OR Operation

Example:

```text
1100 OR 1010
```

Result:

```text
1110
```

---

### XOR Operation

Example:

```text
1100 XOR 1010
```

Result:

```text
0110
```

---

### NOT Operation

Example:

```text
1100
```

```text
0011
```

Every bit flips.

## `ALU Control Signals`

`The ALU can perform many operations.`

How does it know which one to execute?

```
ALU

├─ Adders
├─ Subtractors
├─ Comparators
├─ AND circuits
├─ OR circuits
├─ XOR circuits
├─ Shift circuits
└─ Multiplexers
```

It receives a control signal.

Example:

```text
Control = ADD
Control = SUBTRACT
Control = AND
Control = OR
```

The control signal tells the ALU what operation to perform.

## ALU And Registers

The ALU never works alone.

It constantly communicates with registers.

Example:

```text
Register A = 5
Register B = 3
```

CPU executes:

```text
ADD
```

Flow:

```text
+-----------+
| Register A|
+-----------+
      |
      v
+-----------+
|    ALU    |
+-----------+
      ^
      |
+-----------+
| Register B|
+-----------+
      |
      v
+-----------+
| Register C|
+-----------+
```

Result:

```text
Register C = 8
```

---

## Real CPU Example

Suppose:

```rust
let total = price + tax;
```

CPU steps:

```text
Load price into Register A
Load tax into Register B
Send A and B to ALU
ALU performs ADD
Store result in Register C
```

This happens billions of times every second.

## First Principles Summary

Registers provide memory.

The ALU provides computation.

Together they create the foundation of a CPU.

The ALU performs:

- Arithmetic.
- Logic.
- Comparisons.

Internally, the ALU is built from:

```text
Logic Gates -> Adders -> Multiplexers -> Control Circuits
```

The ALU does not understand mathematics.

It simply manipulates bits according to rules.

---

## Key Takeaways

- ALU stands for Arithmetic Logic Unit.
- The ALU is the calculator of the CPU.
- It performs arithmetic operations.
- It performs logical operations.
- It performs comparisons.
- It receives data from registers.
- It stores results back into registers.
- It uses control signals to select operations.
- It generates status flags.
- Every modern CPU depends on an ALU.

## Solution

1. What does ALU stand for?
- Arithmetic Logic Unit.

2. Why does a CPU need an ALU?
- To perform arithmetic and logical operations on data (adds, compares, bitwise ops, etc.).

3. What is the difference between arithmetic and logical operations?
- Arithmetic operates on numeric values (+, −, ×, ÷); logical operates on bits (AND, OR, XOR, NOT).

4. How does the ALU perform addition?
- By using chains of full adders (bitwise adders that propagate carries) to compute multi-bit sums.

5. What role do Full Adders play?
- Each full adder adds one bit plus a carry-in and produces a sum bit and a carry-out; they are the building blocks of multi-bit addition.

6. What are control signals?
- Signals that tell the ALU which operation to perform (add, subtract, AND, OR, etc.).

7. Why does the ALU use multiplexers?
- To select the correct operation output (from adders, logic units, shifters) based on control signals.

8. What is a Flags Register?
- A small register that stores status bits (zero, carry, overflow, sign) produced by ALU operations.

9. What is the Zero Flag?
- A flag set when the ALU result is zero.

10. How do registers and the ALU work together?
- Registers supply operands to the ALU; the ALU computes results, which are then written back into registers.
