**Before Codd:** 50 airlines = 50 custom pointer programs

**Codd's goal:** 1 universal math equation = works for ALL airlines

1. Set Theory = "Math Groups, No Duplicates"

SET THEORY: Group of unique things. No Steve twice.

**Koel Example:**
SET A: users = {Steve, Priya}
SET B: flights = {DL456, DL457}
SET C: bookings = {(Steve, DL456), (Priya, DL456)}

**Question: "DL456 passengers?"**
Math: {x | (x, DL456) ∈ bookings} = {Steve, Priya}
Computer: 1ms. No custom code.

**Your Koel table:**

```text
| id | name |
|----|-------|
| 1 | Steve |
| 2 | Priya |
| 3 | Aariv |
```

**Why?** If Steve registers twice → 2 Steve rows = broken math.
**Primary key** = math guarantee: "Steve appears exactly once"

2.Predicate Logic = "Ask Questions in Plain English"

    - Old way: "Follow Steve pointer → Flight pointer → Seat pointer"
    - **Codd's way:** "Show me flights WHERE user = Steve"
    - **Koel SSO Example:**
    "Show sessions WHERE user_id = 1 AND product_domain = 'sari.one'"
    → Result: Steve's Sari session ✅
    - **Plain English → Computer math:**
    WHERE = "filter this set"
    JOIN = "combine these sets"

3.Tables = Relations (Math Term)

- **Codd's math name:** "Relation" = table (set of tuples)
- **Why "relation"?** Math function: user_id → name (1:1 mapping)

TABLE users (RELATION):
| user_id | name |
|---------|-------|
| 1 | Steve |
| 2 | Priya |

Math: f(1) = "Steve", f(2) = "Priya"

4.Rows = Tuples, Columns = Attributes

- **Math terms → Simple English:**
- ROW (tuple) = 1 fact: "Steve has email steve@gmail.com"
- COLUMN (attribute) = 1 property type: "email" column

TABLE users:

```text
| id | **name** (ATTRIBUTE) | **email** (ATTRIBUTE) |
|----|---------------------|----------------------|
| 1 | Steve (TUPLE) | steve@gmail.com |
| 2 | Priya (TUPLE) | priya@gmail.com |
```
