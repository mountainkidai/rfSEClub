# Phase 0 — Computational Thinking

## Introduction

Programming is not writing code.

Programming is solving problems.

Rust, Python, JavaScript, Go, and C++ are merely tools used to express solutions.

Before learning any programming language, an engineer must first learn how to think.

This phase develops the mental models that every great engineer uses to understand problems, design systems, automate work, and model reality.

By the end of this phase, students should be able to look at any real-world problem and break it into smaller components that a computer can execute.

# 0.15 Final Project — Design Before Coding

Students must design systems without writing code.

Examples:

- ATM
- Google Maps
- Instagram
- Uber
- Amazon
- Phunsuk
- Sari

For each system:

1. Define the problem
2. Define inputs
3. Define outputs
4. Define state
5. Define entities
6. Define relationships
7. Define algorithms
8. Define tradeoffs
9. Define failure scenarios

Only after mastering this phase should students move to Rust.

Because coding is not engineering.

Coding is merely one tool used by engineers.
