# Phase 01 — Types of Problems

# Introduction

In Phase 00 we learned:

```text
Current State
↓
Desired State
↓
Gap
↓
Problem
```

A problem exists whenever reality differs from what we want reality to be.

But not all problems are the same.

A broken website.

A dying patient.

A traffic jam.

A bankrupt company.

A war.

A climate crisis.

All are problems.

Yet they require completely different approaches.

One of the biggest mistakes engineers make is treating every problem as the same type of problem.

This leads to:

```text
Wrong Problem Type
↓
Wrong Solution
↓
Wasted Time
↓
Failure
```

Before solving a problem, we must first classify it.

This phase teaches you how to see problems the way elite engineers, scientists, founders, investors, and civilization builders see them.

The goal is simple:

```text
Observe Problem
↓
Classify Problem
↓
Understand Constraints
↓
Choose Correct Framework
↓
Solve Efficiently
```

# 1.1 Why Problem Classification Matters

## Why This Matters

Different problems require different tools.

You do not use surgery to fix a software bug.

You do not use a software engineer to fix a broken bone.

You do not use marketing to solve a physics problem.

The first step in solving any problem is understanding what kind of problem it actually is.

# Outcome

By the end of Phase 01 you should understand:

```text
Problems
↓
Categories
↓
Constraints
↓
Frameworks
↓
Solutions
```

and be able to answer:

- What type of problem is this?
- What domain does it belong to?
- Is it simple or complex?
- Is it technical or social?
- Is it local or systemic?
- What framework should be used?
- What constraints matter most?
- What kind of solution is required?

Most importantly:

```text
Bad Engineers
Jump To Solutions

Good Engineers
Define Problems

Elite Engineers
Classify Problems

Civilization Builders
Understand Systems
```

Because before solving a problem, you must first understand what kind of problem you are actually looking at.
