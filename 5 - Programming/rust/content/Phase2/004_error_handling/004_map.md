# `map()`

## First Principle

Suppose someone gives you a number.

```text
5
```

You want to double it.

```text
5

↓

10
```

Simple.

Now suppose the number is inside an `Option`.

```text
Some(5)
```

How do you double it?

You could write

```rust
match number {
    Some(value) => Some(value * 2),

    None => None,
}
```

Works.

But Rust gives us something much shorter.

---

# What does `map()` do?

`map()` means

> **If there is a value, transform it.**

If there isn't,

Leave it alone.

---

## Visualization

```text
Some(Value)

↓

Transform

↓

Some(New Value)
```

or

```text
None

↓

Nothing Happens

↓

None
```

---

# Example

```rust
let number = Some(5);

let doubled = number.map(|x| x * 2);

println!("{:?}", doubled);
```

Output

```text
Some(10)
```

Notice

We didn't write `match`.

---

# What if it's None?

```rust
let number: Option<i32> = None;

let doubled = number.map(|x| x * 2);

println!("{:?}", doubled);
```

Output

```text
None
```

Nothing crashes.

Nothing executes.

---

# Think of a Factory

Imagine

```text
Apple

↓

Juice Machine

↓

Apple Juice
```

Now imagine

```text
Nothing

↓

Juice Machine

↓

Nothing
```

The machine only works if something enters it.

That's exactly how `map()` works.

# Breaking Down

```rust
number.map(|x| x * 2)
```

Read it as

> If there is a number,

call it

```text
x
```

then return

```text
x * 2
```

---

# What is `|x|`?

This is called a **closure**.

Don't worry about closures yet.

For now,

just think

```text
|x|

↓

Take the value

↓

Call it x
```

Exactly like

```rust
fn double(x: i32) -> i32 {
    x * 2
}
```

# Works on Result Too

Remember

```rust
Result

↓

Ok

Err
```

`map()` only changes

```text
Ok
```

Example

```rust
let score: Result<i32, String> =
    Ok(50);

let doubled =
    score.map(|x| x * 2);
```

Output

```text
Ok(100)
```

---

If

```rust
Err("Database Error")
```

then

```rust
.map(...)
```

does nothing.

Output

```text
Err("Database Error")
```

The error passes through unchanged.

```rs
fn main() {
    // Success case
    let score: Result<i32, String> = Ok(50);

    let doubled = score.map(|x| x * 2);

    println!("{:?}", doubled); // Ok(100)

    // Error case
    let score: Result<i32, String> =
        Err("Database Error".to_string());

    let doubled = score.map(|x| x * 2);

    println!("{:?}", doubled); // Err("Database Error")
}
```

---

# Visualization

```text
Result

        │

   ┌────┴────┐

   │         │

Ok(5)     Err(Error)

   │

map()

   │

Ok(10)
```

Notice

The error side is untouched.

# Why is map Useful?

Without map

```rust
match value {

    Ok(v) => Ok(v * 2),

    Err(e) => Err(e),
}
```

With map

```rust
value.map(|v| v * 2)
```

Cleaner.

# Rule to Remember

- `map()` transforms the success value.
- `Option` → transforms `Some`.
- `Result` → transforms `Ok`.
- `None` and `Err` pass through unchanged.
- `map()` replaces many simple `match` statements.

# Solutions

## Solution 1

```rust
let number = Some(8);

let doubled =
    number.map(|x| x * 2);
```

Output

```text
Some(16)
```

---

## Solution 2

```rust
let name =
    Some(String::from("Steve"));

let upper =
    name.map(|x| x.to_uppercase());
```

Output

```text
Some("STEVE")
```

---

## Solution 3

Output

```text
None
```

Because there is no value to transform.

---

## Solution 4

Output

```text
Ok(15)
```

`map()` transforms the `Ok` value from `10` to `15`.

---

## Solution 5

Output

```text
Err("Failed")
```

The error is left unchanged.

---

## Solution 6

```rust
let result =
    number.map(|x| x * 3);
```

This is equivalent to the original `match`.

---

## Solution 7

`map()` only transforms an existing value.

- `Option` transforms only `Some`.
- `Result` transforms only `Ok`.

If the value is `None` or `Err`, there is nothing to transform, so Rust simply returns it unchanged.

# The Big Picture

Notice the progression:

```text
Option<T>

↓

Some / None
```

```text
Result<T, E>

↓

Ok / Err
```

```text
?

↓

Stop on error
```

```text
map()

↓

Transform success values
```

This is why you'll often see code like:

```rust
let email = find_user()
    .map(|user| user.email)?;
```

You can now read it as:

1. Find the user.
2. If successful, extract the email.
3. If an error occurs, return it immediately.

That's concise, expressive Rust.

### Next Concept

The next concept is **`map_err()`**.

It's the mirror image of `map()`: instead of transforming the **success value (`Ok`)**, it transforms the **error value (`Err`)**. This is used constantly in Axum and SQLx to convert one error type into another.
