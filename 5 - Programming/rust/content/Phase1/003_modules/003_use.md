# `use`

## First Principle

Imagine your friend Steve lives in another city.

Without contacts, every time you call him, you must write his full address.

```text
India

↓

Himachal Pradesh

↓

Shimla

↓

Jakhu

↓

Steve
```

That's long.

Instead, you save him in your contacts.

Now you simply write

```text
Steve
```

The contact app remembers the full address.

That's exactly what `use` does.

---

# Without `use`

Suppose you have

```rust
mod math;

fn main() {
    let answer = math::add(5, 10);

    println!("{}", answer);
}
```

Every time

```rust
math::add()
```

you write the module name.

---

# Imagine a Bigger Project

```rust
auth::login();

auth::logout();

auth::verify();

auth::refresh();

auth::signup();
```

Repeating

```text
auth::
```

again and again becomes annoying.

---

# Using `use`

Instead

```rust
use auth::login;

fn main() {
    login();
}
```

Notice

Before

```rust
auth::login();
```

After

```rust
login();
```

Much cleaner.

---

# What does `use` actually do?

It **brings a name into the current scope.**

Think of it as creating a shortcut.

```text
Before

auth::login()

──────────────

After

login()
```

---

# Real Example

Suppose

```rust
mod math;
```

Inside

```rust
// math.rs

pub fn add(a: i32, b: i32) -> i32 {
    a + b
}
```

Instead of

```rust
fn main() {
    println!("{}", math::add(5, 10));
}
```

Write

```rust
use math::add;

fn main() {
    println!("{}", add(5, 10));
}
```

# Importing Multiple Items

Suppose

```rust
mod math;
```

contains

```rust
pub fn add() {}

pub fn subtract() {}

pub fn multiply() {}
```

Instead of

```rust
use math::add;
use math::subtract;
use math::multiply;
```

Rust allows

```rust
use math::{
    add,
    subtract,
    multiply,
};
```

Cleaner.

---

# Importing Everything

Sometimes

```rust
use math::*;
```

means

> Bring everything public from `math`.

Useful sometimes.

Not always recommended.

---

# Why?

Imagine

```rust
math::add()

physics::add()
```

Now if you import everything

```rust
use math::*;
use physics::*;
```

Rust doesn't know which

```rust
add()
```

you mean.

Explicit imports are usually better.

---

# Real Rust Examples

You'll constantly see

```rust
use std::collections::HashMap;
```

Instead of

```rust
std::collections::HashMap::new();
```

You can write

```rust
HashMap::new();
```

---

Another example

Instead of

```rust
std::string::String::new()
```

you usually write

```rust
String::new()
```

because `String` is already in scope.

# Real Koel Example

Without `use`

```rust
axum::Json(...)

axum::extract::State(...)

sqlx::PgPool
```

With

```rust
use axum::{
    Json,
    extract::State,

use sqlx::PgPool;
```

```rust
async fn login(

```

Much easier to read.

# `use` Doesn't Copy Anything

This is important.

```text
Original

↓

Still Exists
```

`use` only creates another way to refer to it.

Think

```text
Shortcut

NOT

Copy
```

---

# Mental Model

```text
crate

│

├── auth

│      │

│      └── login()

│

└── main
```

Without `use`

```text
main

↓

auth

↓

login()
```

With

```text
main

↓

login()
```

Rust remembers where it came from.

---

# Rule to Remember

- `use` creates a shortcut.
- It makes code cleaner.
- It does **not** move or copy anything.
- It only works for items that are visible (`pub`).

# Solutions

## Solution 1

```rust
use math::add;

fn main() {
    add(5, 10);
}
```

---

## Solution 2

```rust
use auth::{
    login,
    logout,
    signup,
};
```

---

## Solution 3

```rust
use std::collections::HashMap;
```

Brings `HashMap` into the current scope, allowing you to write `HashMap` directly instead of `std::collections::HashMap`.

---

## Solution 4

No.

`use` does **not** copy anything.

It creates a shortcut (an alias in the current scope) to an existing item.

---

## Solution 5

```rust
use std::collections::HashMap;

fn main() {
    let map = HashMap::new();
}
```

---

## Solution 6

Generally,

```rust
use math::{add, subtract};
```

is better because:

- It makes dependencies explicit.
- It avoids name conflicts.
- It improves readability.
- It's easier to maintain in large codebases.

`use math::*;` is convenient but can lead to ambiguity when multiple modules export items with the same name.

---

## Solution 7

No.

If the function is:

```rust
fn login() {}
```

it is private to its module.

Only `pub` items can be imported from another module using `use`.
